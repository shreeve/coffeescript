# CS3 const/let Proof of Implementation

## How to Test

Use the `--cs3` flag to enable CS3 compilation with ES6 output:

```bash
coffee -c --cs3 script.coffee
```

## Key Differences: CS2 vs CS3

### CS2 (Traditional)
- Uses `var` for all variable declarations
- Variables are function-scoped
- All variables hoisted to top of function

### CS3 (Modern ES6)
- Uses `const` for functions and classes (immutable bindings)
- Uses `let` for regular variables (block-scoped)
- Uses `for...of` loops instead of traditional for loops

## Example Comparison

### Input CoffeeScript:
```coffee
class MyClass
  constructor: ->

add = (a, b) -> a + b

x = 5
x = x + 1

for i in [1, 2, 3]
  console.log i
```

### CS2 Output (using `var`):
```javascript
// Generated by CoffeeScript 2.7.0
(function() {
  var MyClass, add, i, j, len, ref, x;

  MyClass = class MyClass {
    constructor() {}
  };

  add = function(a, b) {
    return a + b;
  };

  x = 5;
  x = x + 1;

  ref = [1, 2, 3];
  for (j = 0, len = ref.length; j < len; j++) {
    i = ref[j];
    console.log(i);
  }
}).call(this);
```

### CS3 Output (using `const`/`let`):
```javascript
let i, ref, x;
const MyClass = class MyClass {
  constructor() {}
};

const add = function(a, b) {
  return a + b;
};

x = 5;
x = x + 1;

ref = [1, 2, 3];
for (i of ref) {
  console.log(i);
}
```

## What Gets `const` vs `let`

### `const` is used for:
- **Functions**: They're never reassigned
- **Classes**: They're never reassigned

### `let` is used for:
- **Regular variables**: They might be reassigned
- **Loop variables**: They need to be accessible after the loop (CoffeeScript semantics)
- **Any variable that gets reassigned**: Like `x = x + 1`

## Implementation Details

1. **ES6 Backend (`src/es6.coffee`)**: Sets `cs3: true` in compile options
2. **Nodes (`src/nodes.coffee`)**: Checks `o.cs3` flag to determine output:
   - `if o.cs3` → use `let`/`const`
   - else → use `var`
3. **Variable Analysis**: Pre-pass analysis determines which variables can be `const`
